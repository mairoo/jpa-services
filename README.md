# 목표
스프링부트 자바 기반 프로젝트 서비스 계층에 트랜잭션 스크립트 패턴과 퍼사드 패턴의 적용

# 수행 작업 및 요건
- 작업 수행 순서
    1. DB 트랜잭션 수행(1)
    2. 비동기 외부 API 1 호출 (DB 트랜잭션 로깅 포함)
    3. 비동기 외부 API 호출결과를 가지고 DB 트랜잭션 수행(2)
    4. 비동기 외부 API 2 호출 (통보 등  DB 트랜잭션 로깅 포함)
- 주요 특징
    - 1~4번은 하나의 원자적 동작
    - 4번의 실패는 1~3번 롤백 없음
    - 3번의 실패는 2번 취소 api 동작과 1번 롤백
    - 2번의 실패는 1번 롤백
